# Concept: Coordinator Delegation Discipline

## Overview
Coordinator Delegation Discipline represents the fundamental principle that Coordinator mode operates as "Conductor, Not Performer" - exclusively orchestrating workflows through delegation to specialist modes rather than performing direct technical work. This concept emerged from critical Learn mode behavioral pattern improvements addressing recurring coordination boundary violations.

## Key Principles

### Anti-Execution Protocol Enforcement
- **FORBIDDEN**: Direct technical analysis, code implementation, file investigation, architectural design
- **REQUIRED**: Systematic delegation to appropriate specialist modes (implement, review-quality, architect, debug)
- **CRITICAL**: Reflexive delegation responses preventing temptation to perform direct work
